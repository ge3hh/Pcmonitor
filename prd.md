# Pcmonitor 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
Pcmonitor - Windows 资源监控工具

### 1.2 产品定位
一款轻量级的 Windows 系统资源监控工具，帮助用户实时了解系统运行状态，优化系统性能。

### 1.3 目标用户
- 需要监控系统资源的技术人员
- 关注系统性能的普通用户
- 需要排查系统问题的开发者

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 CPU 监控
- **功能描述**: 实时显示 CPU 使用率
- **监控指标**:
  - 总体 CPU 使用率
  - 各核心使用率
  - CPU 频率
  - CPU 温度（如硬件支持）
- **展示方式**: 实时图表 + 数字显示

#### 2.1.2 内存监控
- **功能描述**: 实时显示内存使用情况
- **监控指标**:
  - 已用内存 / 总内存
  - 可用内存
  - 内存使用率百分比
  - 虚拟内存使用情况
- **展示方式**: 进度条 + 数字显示

#### 2.1.3 磁盘监控
- **功能描述**: 监控磁盘使用情况和 I/O 性能
- **监控指标**:
  - 各分区容量使用情况
  - 磁盘读写速度
  - 磁盘 I/O 等待时间
- **展示方式**: 饼图/柱状图 + 实时速率显示

#### 2.1.4 网络监控
- **功能描述**: 监控网络流量和连接状态
- **监控指标**:
  - 上传/下载速度
  - 总流量统计
  - 网络连接数
  - 各进程网络使用情况
- **展示方式**: 实时折线图 + 速率显示

#### 2.1.5 GPU 监控（可选）
- **功能描述**: 监控显卡使用情况
- **监控指标**:
  - GPU 使用率
  - 显存使用情况
  - GPU 温度
- **展示方式**: 图表 + 数字显示

### 2.2 扩展功能

#### 2.2.1 进程管理
- 显示当前运行的进程列表
- 按 CPU/内存/磁盘/网络使用率排序
- 支持结束进程
- 进程详细信息查看

#### 2.2.2 告警功能
- 可设置资源使用阈值
- 超过阈值时弹出通知
- 支持声音提醒

#### 2.2.3 历史记录
- 保存资源使用历史数据
- 支持查看历史趋势图表
- 数据导出功能

#### 2.2.4 开机自启
- 支持设置开机自动启动
- 最小化到系统托盘

## 3. 非功能需求

### 3.1 性能要求
- 自身资源占用低（CPU < 1%，内存 < 50MB）
- 数据采集频率可配置（默认 1 秒）
- 界面刷新流畅，无卡顿

### 3.2 兼容性
- 支持 Windows 10/11
- 支持 x64 架构
- 适配不同分辨率屏幕

### 3.3 用户体验
- 界面简洁直观
- 支持深色/浅色主题
- 窗口大小可调整
- 支持置顶显示

## 4. 技术方案

### 4.1 技术栈
- **开发语言**: Python 3.8+
- **GUI 框架**: PyQt5 / Tkinter / PySide6
- **系统信息采集**: psutil / wmi / pywin32
- **图表展示**: pyqtgraph / matplotlib

### 4.2 架构设计
```
Pcmonitor/
├── core/               # 核心监控模块
│   ├── cpu_monitor.py
│   ├── memory_monitor.py
```

## 6. 开发计划

### 6.1 里程碑

| 阶段 | 时间 | 目标 |
|------|------|------|
| MVP | 第 1 周 | 基础监控功能（CPU/内存/磁盘/网络） |
| V1.0 | 第 2-3 周 | 完善 UI、进程管理、历史记录 |
| V1.1 | 第 4 周 | 告警功能、主题切换、性能优化 |
| V2.0 | 第 5-6 周 | GPU 监控、数据导出、高级设置 |

### 6.2 优先级
- **P0**: CPU、内存、磁盘、网络基础监控
- **P1**: 进程管理、历史记录、告警
- **P2**: GPU 监控、数据导出、主题切换
- **P3**: 插件系统、远程监控

## 7. 验收标准

### 7.1 功能验收
- [ ] 能正确显示 CPU、内存、磁盘、网络使用率
- [ ] 图表实时更新，无卡顿
- [ ] 进程列表准确，可结束进程
- [ ] 告警功能正常工作

### 7.2 性能验收
- [ ] 自身 CPU 占用 < 1%
- [ ] 内存占用 < 50MB
- [ ] 启动时间 < 3 秒

## 8. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 系统 API 兼容性 | 高 | 使用成熟的 psutil 库，充分测试 |
| 性能优化困难 | 中 | 采用异步采集，优化图表渲染 |
| 界面卡顿 | 中 | 使用多线程，降低刷新频率 |
